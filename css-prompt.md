# 临时的prompt， 用于优化css样式， 一次性的

【强约束（必须遵守）】

1. 配色限制（简洁优先）
- 主色 ≤ 1 个（蓝色系）
- 辅助色 ≤ 2 个（森林绿 / 海洋青）
- 功能色固定：
  - 上涨
  - 下跌
  - 警告
- 总颜色数（不含灰度）≤ 6
- 禁止渐变滥用，仅允许在以下场景使用：
  - K线背景弱渐变
  - hover微弱过渡
- 禁止高饱和度大面积使用

2. 灰度体系（必须建立）
- 至少 6 阶灰度（用于背景层级）
- 示例（必须输出完整表）：
  - bg-1（主背景）
  - bg-2（卡片）
  - bg-3（浮层）
  - border
  - text-primary
  - text-secondary

3. 导航尺寸约束（必须小）
- 顶部导航高度：40px ~ 48px
- 侧边导航宽度：56px（图标模式）或 ≤ 180px（压缩模式）
- 导航内容限制：
  - 仅保留核心入口（≤ 6 个）
  - 禁止多层嵌套（最多2层）
- 字体：
  - 导航字号 ≤ 13px
- 间距：
  - item padding ≤ 12px

4. 空间密度（交易场景）
- 优先展示数据区（≥70%屏幕）
- 控制留白，不做大块装饰留白
- 卡片 padding ≤ 16px

5. 对比度规则
- 关键数据（价格/涨跌）对比度 ≥ 4.5
- 次级信息降低 30% 对比度
- 禁止使用纯黑 (#000) 或纯白 (#FFF) 作为大面积背景

新增要求：
1. 必须输出“颜色使用数量统计”，证明符合 ≤6 色限制
2. 必须标注导航最终尺寸（高度/宽度/字号）
3. 必须说明删除了哪些非必要颜色或导航项
