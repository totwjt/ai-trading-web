你正在参与开发一个股票量化交易Web平台。

必须遵守以下约束：

1. 所有行情数据来自本地PostgreSQL，不允许调用外部API
2. 回测引擎为Backtrader
3. 策略必须符合 Backtrader 类结构
4. 回测分为两种模式：
   - 调试模式：同步、小数据量
   - 回测模式：异步、大数据量、实时推送
5. 初期可使用 Backtrader 返回的K线图，后期所有K线数据由后端提供JSON，前端绘制
6. 策略禁止访问系统资源（文件/网络/进程）

补充说明（2026-03-21）：

7. “编译运行 / 预览”仅用于快速验证代码是否正确，允许压缩到近 365 天、小样本股票，以便尽快返回日志和测试结果
8. “运行回测”才是最终回测框架结果，必须使用真实 PostgreSQL 数据并把结果落库保留
9. 当前已确认的真实数据表：
   - `stock_daily`：股票日线 OHLCV
   - `stock_adj_factor`：复权因子
   - `stock_daily_basic`：PE/PB/换手率/市值等基础因子
   - `stock_factor_pro`：日线、复权价格及技术指标因子集合
   - `index_daily`：指数日线，可用于沪深300等基准
   - `trade_calendar`：交易日历
   - `stock_basic`：股票基础信息
10. 最终回测如果继续使用随机样本数据，只能视为调试/预览，不可视为真实回测结果

你的任务是：
- 生成可运行的策略代码
- 设计回测接口
- 优化执行性能
- 保证代码安全性
